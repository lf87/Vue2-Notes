2. Vue instance:

  • 1 instance for entire app, or multiple instances for multiple parts of the app
  • Takes an object as a parameter
<div id="vue-app">
    <h1>Hey, {{ name }}</h1>
</div>

new Vue({
    el: '#vue-app',
    data: {
        name: 'Shaun'
    }
});

3. Data and methods:

  • Properties set in data are the initial values
  • Vue proxies data objects to top of the instance, so they can all be accessed by doing this.xxxx rather than this.data.xxxx
  • Template calls for methods done in the same way as JS likeThis()
  • Only evaluated when explicitly called
  • Used to perform actions on properties
<div id="vue-app">
    <h1>Hey, {{ name }}</h1>
</div>

new Vue({
    el: '#vue-app',
    data: {
        name: 'Shaun'
    }
});

4. Data binding:

  • Use v-bind:attr="data" to bind data properties etc. (can just use a colon) v-html="data" to render html
<div id="vue-app">
    <h1>Data Binding</h1>
    <p><a v-bind:href="website">The Net Ninja Website</a></p>
    <p v-html="websiteTag"></p>
    <p><input type="text" v-bind:value="name" /></p>
</div>

new Vue({
	el: '#vue-app',
	data: {
	    name: 'Shaun',
	    job: 'Ninja',
	    website: 'http://www.thenetninja.co.uk',
	    websiteTag: 'The Net Ninja Website'
	},
	methods: {
	    greet: function(time){
	        return 'Good ' + time + ', ' + this.name;
	    }
	}
});

5. Events:

  • v-on:event is used as an event listener
  • Code can be run within the template or called as external function (method)
  • Parenthesis aren't needed when calling a function i.e. thisFunctionCall() would instead be thisFunctionCall
  • paramaters can be passed in the template in the normal way likeThis(val, 2)
  • Shorthand for v-on is @
<div id="canvas" v-on:mousemove="updateXY">{{ x }} , {{ y }}</div>

new Vue({
    data: {
        x: 0,
        y: 0
    },
    methods: {
        updateXY: function(event){
            this.x = event.offsetX;
            this.y = event.offsetY;
        }
    }
});

6. Events Modifiers:

  • An example of a simple event modifier would be adding .once to the end of a click event. Which would look like this - v-on:click.once
  • .prevent is another example, which works the similar to preventDefault in jQuery

Ref: https://vuejs.org/v2/guide/events.html

7. Keyboard Events

  • JS keyboard events can be chained like so - v-on:keyup.enter.alt="runFunc" or v-on:tab.enter.alt="runFunc"

8. Two-Way Data Binding

  • v-model="name" can be equal to any variable or property
  • The data goes back and forth from the view to the model then back to the model

9. Computed Properties

  • Computed values are cached and therefore efficient
  • Used for manipulating properties (such as large data sets)
  • To run a computed property remove the parenthesis

Explanation on the difference between Methods and Computed properties here - https://stackoverflow.com/questions/44350862/method-vs-computed-in-vue

10. Dynamic CSS Classes

  • Classes can be set based on the value of a boolean
<div v-bind:class="{red: true, blue: true}">
    This would have the class of red and blue
</div>

<div v-on:click="available = !available" v-bind:class="{available: available}">
    Clicking this will toggle the avialable class
</div>

Below is an example of computed properties returned in an element. They will wait for the data to change (using an event listener in this example) and then update.

<button v-on:click="nearby = !nearby">Toggle nearby</button>
<button v-on:click="available = !available">Toggle available</button>
<div v-bind:class="compClasses">
    <span>Ryu</span>
</div>

new Vue({
    el: '#vue-app',
    data: {
        available: false,
        nearby: false
    },
    methods: {

    },
    computed: {
        compClasses: function(){
            return {
                available: this.available,
                nearby: this.nearby,
            }
        }
    }
});

11. Conditionals

In the code below v-if won't show if false. v-else-if will only run when v-if is false

<p v-else--if="success"></p>

v-show Applies the same principal as v-if except it adds a display none, rather than removing the element completely

12. Looping with v-for

  • Vue will know not to render the <template> tag, only the contents within it

Below is an array and an object will be looped through using examples further down.

new Vue({
    el: '#vue-app',
    data: {
        characters: ['Mario', 'Luigi', 'Yoshi', 'Bowser'],
        ninjas: [
            { name: 'Ryu', age: 25 },
            { name: 'Yoshi', age: 35 },
            { name: 'Ken', age: 55 }
        ]
    }
});

This will display all characters in a list.

<ul>
    <li v-for="character in characters">{{ character }}</li>
</ul>

This will display all ninjas with their respective index value.

<ul>
    <li v-for="(ninja, index) in ninjas">{{ index }} . {{ ninja.name }} - {{ ninja.age }}</li>
</ul>

This will output all key pair vlaues in the ninja object, regardless of the contents.

<template v-for="ninja in ninjas">
    <div v-for="(val, key) in ninja">
        <p>{{key}} - {{ val }}</p>
    </div>
    <hr />
</template>

14. Multiple Vue Instances

  • You could use 2 instances if you used them as two independent widgets on your application. I.e. So Vue isn't controlling the whole app.﻿
  • Work the same as if you're working with one instance, they just need to be stored in a variable if you want to access them independantly (see examples below)
  • properties can be maniupulated from outside instances i.e. if you were to do instanceName.dataProp = "Changed from the outside" - This would take priority over what is defined in both instances

Vue can communicate between the 2 instances. In the function changeTitle (below), the line of code one.title = 'Title Changed'; will update the title of instance one from instance two

<div id="vue-app-one">
    <h2>{{ title }}</h2>
    <p>{{ greet }}</p>
</div>
<div id="vue-app-two">
    <h2>{{ title }}</h2>
    <p>{{ greet }}</p>
    <button v-on:click="changeTitle">Change App One Title</button>
</div>
        
var one = new Vue({
    el: '#vue-app-one',
    data: {
      title: 'Vue App One'
    },
    computed: {
      greet: function(){
        return 'Hello, from app one :)';
      }
    }
});

var two = new Vue({
    el: '#vue-app-two',
    data: {
      title: 'Vue App Two'
    },
    computed: {
      greet: function(){
        return 'Yo dudes, this is app 2 speaking to ya';
      }
    },
    methods: {
      changeTitle: function(){
        one.title = 'Title Changed';
      }
    }
});

two.title = 'Changed from outside';

15. Intro to Components

  • Components allow us to create re-usable templates and functionality snippets within different sections of our website.
  • Work very similar to Vue instances
  • Components are output by using the component name in tag format <componentName>like this</componentName>
  • Data can be passed through, but it must be a function not a direct object (Example below). This is so that when the fuincton is called a fresh copy of the data is used i.e. all data objects aren't shared
<div id="vue-app-one">
    <h1>Vue App One</h1>
    <greeting></greeting>
</div>
<div id="vue-app-two">
    <h1>Vue App Two</h1>
    <greeting></greeting>
</div>
        
Vue.component('greeting', {
    template: '<p>Hey there, I am {{ name }}. <button v-on:click="">Change name</button></p>',
    data: function(){
        return {
            name: 'Yoshi'
        }
    },
    methods: {
        changedName: function() {
            this.name = 'Mario';
        }
    }
});

new Vue({
    el: '#vue-app-one'
});

16. Refs

  • We can use refs to reach into the DOM and access information such as text input value's, inner HTML and more
  • A ref attribute is added to an element to make it accessible

In the example below, when the submit button is hit, it'll run the readRefs function, return the value from the object created by the input ref(erence), and update the output.

<div id="vue-app-one">
	<input type="text" ref="input" />
    <button v-on:click="readRefs">Submit</button>
    <p>{{ output }}</p>
</div>

new Vue({
    el: '#vue-app-one',
    data: {
        output: 'Your favourite food'
    },
    methods: {
        readRefs: function(){
            console.log(this.$refs);
            this.output = this.$refs.input.value;
        }
    }
});

18. Vue Files & The Root Component

Inside the new Vue instance in main.js we render the app using render:h => h(App) (ES6 arrow function). This takes the App.vue file, looks for the property/ID assigned to el, and renders it to our root component, at the very top of out application.

  • A Vue file is just an extension of a component
  • Vue files contain template, script, and style tags
  • Data still needs to be ran as a function, but in ES6 data function() is the same as data()
  • In the template tags we must have a single element (e.g. div or section) to wrap around all children elements

Below is an example of App.vue:

<template>

</script>

19. Nesting Components

For a large scale app, our component structure might look something like this:

  • Components must be registered globally or locally

Globally registered would look like this:

import Ninjas from './Ninjas.vue'
Vue.component('ninjas', Ninjas);

We can then register it in our Vue file by creating a HTML tag matching the component name. So in this case the element would be called <ninjas></ninjas>.

To nest a component locally, you would import the same way as previously, but also register a component. It would look like this:

import Ninjas from './Ninjas.vue';
export default {
    components: {
        'ninjas': Ninjas
    },
    data () {
        return {
          title: 'Ninja App'
        }
    }
}

22. Props

A way to pass data from one component to another (parent to child)

If you wanted to display the same set of data on 5 pages, you would declare this data in a component such as your root component and then pass the data down to its children using props

This is how we would receieve the data with props:

export default {
    props: ['ninjas']
}

This is how we would pass the data down to the child component (inside App.vue):

<app-ninjas v-bind:ninjas="ninjas"></app-ninjas>

v-bind makes it dynamic, without this Vue would see this is a string

Props can then be accessed inside methods, as if they were declared in the same component:

methods: {
    test.function(){
        this.ninjas
    }
}

Props can also be validated. Becuase the prop in this example is an array, we can check for it by doing the following:

export default {
    props: {
        ninjas: {
            type: array,
            required: true
        }
    }
}

23. Primitive vs Reference Types

References passed through as a prop to another component is literally just passing through a reference to that data, not a new set of data. Therefore if this data is edited, it will indirectly edit the original data. Say for example you had 1 method for deleting an element on a page, if the component being used on that page has an event handler on it, it will trigger the event on both instances of the component.

With primitive types you could pass down a props from your root component in to your header and footer file. An example might be a string of 'title' reading 'Website'. In the header file you could assign an event handler calling a function called changeTitle (remember no parenthesis for methods) and it would only update it for the specific component.

  • Reference types are objects, and arrays
  • Primitive types are strings, booleans, and numbers

24. Events (child to parent)

This kind of reactive relation is useful to know due to the fact that reference types only update the current component.

  • Using this.$emit will emit an event on this component

Below is an example of how to emit an event. The first parameter is the event you're interacting with. The 2nd parameter is the data you want to pass with the event:

methods: {
    changeTitle: function() {
    this.$emit('changeTitle', 'Vue Wizards');
    }
}

We will then tell our root component to listen for this event using v-on, it can then fire a function of its own, and then we can retreieve the data (2nd paremeter or $emit) by taken through $event as a parameter:

<app-header v-bind:title="title" v-on:changeTitle="updateTitle($event)">
</app-header>

Next we will create the updatedTitle function, which will be used to retrieve the event we passed along in to our root component:

methods: {
    updatedTitle: function(updatedTitle) {
    this.title = updatedTitle;
    }
}

25. The Event Bus

The event bus is a Vue instance, which can emit and listen to events. It allows for direct component to component data transfer

In this example, a new event bus is created and exported so it can be used in other components.

export const bus = new Vue();

Importing it in a component can be done like this (notice the exclusion of the file extension):

import { bus } from '../main';

Previously, with the parent to child event, we were emitting the event on the current component. Instead we want to emit it on the event bus, this can be acheieved like so:

methods: {
    changeTitle: function() {
        this.title = 'Vue Wizards'; // edit title directly
        bus.$emit('titleChanged', 'Vue Wizards');
    }
}

In another component we could listen for that event. This needs to be done after the event bus is created, which is at the 'created' stage of the life Vue life cycle. A flat arrow callback function is then used to get the data (2nd parameter) to change this.title:

created(): {
    changeTitle: function() {
        bus.$on('titleChanged', (data) => {
            this.title = data
        })
    }
}

27. Slots

Similarily with props, this will exlpain how to pass something more complex such as a HTML template down to child component using the <slot> tag

In our root component (App.vue) we might have something that looks like this:

<template>
    <div>
        <form-helper>
            <div slot="form-header">
                <h3>This is the title of a form</h3>
                <p>This is some info about the form</p>
            </div>
        </form-helper>
    </div>
</template>

And in formHelper.vue we would access this markup using the <slot> tag:

<template>
    <div>
        <h1>I am the form helper</h1>
        <slot></slot>
    </div>
</template>

Slots parts can be accessed individually by naming them i.e. <h2 slot="heading"></h2>, and accessed like so - <slot name="heading"></slot>

  • Data can be added, and slots can be styled in the same component file where the slot is declared

28. Dynamic Components

Dynamic components allow us to dynamically change which component is output to the browser. This can be acheieved by using the <component> tag, which we can then bind data to to change the state of it.

When switching from one component to another, by default, any manipulated data will be destroyed/reset. By wrapping our component tag in a <keep-alive> tag, we can maintain the state of that component.

This is a basic example of dynamic components in action:

<template>
    <div>
        <keep-alive>
            <component v-bind:is="component-name"></component>
        </keep-alive>
        <button v-on:click="component = 'form-one'">Show form one</button>
        <button v-on:click="component = 'form-two'">Show form two</button>
    </div>
</template>

<script>
    import formOne from './components/formOne.vue';
    import formTwo from './components/formTwo.vue';
    export default {
        components: {
            'form-one': formOne,
            'form-two': formTwo
        },
        data () {
            return {
                component: 'form-one'
            }
        },
        methods: {
            handleSubmit: function(){
                alert('thanks for submitting');
            }
        }
}
</script>

32. HTTP Requests

HTTP Requests can be made using the vueResource plugin

First the plugin needs inporting and initialising from your main.js file

import Vue from 'vue'
import VueResource from 'vue-resource'
import App from './App.vue'

// Use vue-resource package
Vue.use(VueResource);

new Vue({
  el: '#app',
  render: h => h(App)
})

Then for example we could create a button, that on click would call a function for making a HTTP post request. We can prevent the default event of the button using the prevent modifier

<button v-on:click.prevent="post">Add Blog</button>

In the function we say this to refer to the component, and then what type of request your're making, followed by where you're posting to.
Next we'd define the properties in the object. Once all that is complete we can add on a then method - this returns what's known as a promise, and won't run until the post has sucessfully completed, it will run whatever function is passed through to the then method
